# prompts.md

A continuación se listan prompts de ejemplo que se pudieron utilizar para construir el documento de diseño de LTI.

## Prompt 1: definición del producto
```text
Actúa como Product Manager y Solution Architect.
Necesito diseñar la primera versión de un ATS llamado LTI (Applicant Tracking System).
El sistema debe destacar por:
- aumentar la eficiencia de HR
- mejorar la colaboración en tiempo real entre recruiters y hiring managers
- incluir automatizaciones
- incorporar asistencia de IA en tareas de recruiting

Quiero una propuesta inicial con:
- descripción breve del producto
- propuesta de valor
- ventajas competitivas
- funcionalidades principales
Todo en español y con tono profesional.
```

## Prompt 2: benchmarking funcional
```text
Investiga qué capacidades diferencian hoy a los ATS modernos más competitivos.
Pon foco en structured hiring, scorecards, automatización, colaboración y uso de IA.
Luego propón qué capacidades debería incluir LTI en su primera versión para competir bien.
Resume en español y con enfoque de producto.
```

## Prompt 3: Lean Canvas
```text
Genera un Lean Canvas para una startup SaaS B2B llamada LTI que desarrollará un ATS moderno.
Incluye:
- problema
- segmentos de clientes
- propuesta de valor única
- solución
- canales
- ingresos
- costes
- métricas clave
- ventaja injusta
Devuélvelo en formato que pueda pasar a Markdown.
```

## Prompt 4: casos de uso
```text
Define los 3 casos de uso principales para la primera versión de un ATS moderno llamado LTI.
Para cada caso de uso incluye:
- objetivo
- actores
- flujo principal
- reglas relevantes
- un diagrama simple compatible con Mermaid
Todo en español.
```

## Prompt 5: modelo de datos
```text
Diseña un modelo de datos inicial para un ATS llamado LTI.
Debe cubrir:
- entidades
- atributos con nombre y tipo
- relaciones entre entidades
Incluye entidades como vacante, candidato, postulación, entrevista, scorecard, usuario, comentarios, automatizaciones e insights de IA.
Devuelve también un ERD en Mermaid.
```

## Prompt 6: arquitectura de alto nivel
```text
Actúa como arquitecto de software.
Diseña la arquitectura de alto nivel para LTI, un ATS con colaboración en tiempo real, automatizaciones e IA.
Incluye:
- explicación de módulos principales
- decisiones arquitectónicas
- diagrama en Mermaid
- justificación de por qué conviene partir con modular monolith o microservicios
```

## Prompt 7: diagrama C4
```text
Genera un diagrama C4 para LTI y profundiza en uno de sus componentes.
Puedes elegir el componente de IA, automatizaciones o colaboración.
Necesito:
- contexto
- contenedores
- componentes
- explicación del componente elegido
Usa Mermaid si es posible y responde en español.
```

## Prompt 8: consolidación final
```text
Con toda la información anterior, arma un único documento Markdown listo para entregar en GitHub.
Debe incluir:
- descripción del software
- valor añadido y ventajas competitivas
- funciones principales
- Lean Canvas
- 3 casos de uso con diagrama
- modelo de datos
- diseño de alto nivel con diagrama
- diagrama C4 con profundidad en un componente
- conclusión
Tono profesional, claro y estructurado.
```
